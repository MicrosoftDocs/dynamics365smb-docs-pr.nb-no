# <a name="flush-components-according-to-operation-output"></a><span data-ttu-id="c5414-103">Lagertrekke komponenter i henhold til operasjonsavgang</span><span class="sxs-lookup"><span data-stu-id="c5414-103">Flush Components According to Operation Output</span></span>

<span data-ttu-id="c5414-105">Hvis en produksjonsordre om å produsere 800 meter for eksempel krever 8 kg av en komponent, bokføres 2 kg automatisk som forbruk når du bokfører 200 meter som avgang.</span><span class="sxs-lookup"><span data-stu-id="c5414-105">For example, if a production order to produce 800 meters requires 8 kg of a component, then when you post 200 meters as output, 2 kg are automatically posted as consumption.</span></span> 

<span data-ttu-id="c5414-106">Denne funksjonaliteten er nyttig av følgende årsaker:</span><span class="sxs-lookup"><span data-stu-id="c5414-106">This functionality is useful for the following reasons:</span></span>  

- <span data-ttu-id="c5414-107">**Lagerverdisetting**</span><span class="sxs-lookup"><span data-stu-id="c5414-107">**Inventory Valuation**</span></span>

    <span data-ttu-id="c5414-108">Verdipostene for avgang og forbruk opprettes parallelt ettersom produksjonsordren går fremover.</span><span class="sxs-lookup"><span data-stu-id="c5414-108">Value entries for output and consumption are created in parallel as the production order progresses.</span></span> <span data-ttu-id="c5414-109">Uten rutekoblingskoder vil lagerverdien øke etter hvert som avgangen bokføres og deretter reduseres på et senere tidspunkt når verdien for komponentforbruk bokføres sammen med den ferdige produksjonsordren.</span><span class="sxs-lookup"><span data-stu-id="c5414-109">Without routing link codes, the inventory value will increase as output is posted and then decrease at a later point in time when the value of component consumption is posted together with the finished production order.</span></span>  
- <span data-ttu-id="c5414-110">**Lagertilgjengelighet**</span><span class="sxs-lookup"><span data-stu-id="c5414-110">**Inventory Availability**</span></span>

    <span data-ttu-id="c5414-111">Med gradvis forbruksbokføring er komponentvarenes tilgjengelighet mer oppdatert, noe som er viktig for å opprettholde den interne balansen mellom behov og forsyning.</span><span class="sxs-lookup"><span data-stu-id="c5414-111">With gradual consumption posting, the availability of component items is more up-to-date, which is important to maintain the internal balance between demand and supply.</span></span> <span data-ttu-id="c5414-112">Uten rutekoblingskoder kan andre behov for komponenten tro at den er tilgjengelig så lenge den venter på en forsinket forbruksbokføring.</span><span class="sxs-lookup"><span data-stu-id="c5414-112">Without routing link codes, other demands for the component may believe that it is available as long as it is pending a delayed consumption posting.</span></span>  
- <span data-ttu-id="c5414-113">**Til rett tid**</span><span class="sxs-lookup"><span data-stu-id="c5414-113">**Just-in-Time**</span></span>

    <span data-ttu-id="c5414-114">Med evnen til å tilpasse produktene til kundens ønsker kan du minimere svinn ved å kontrollere at arbeids- og systemendringer bare forekommer når det er nødvendig.</span><span class="sxs-lookup"><span data-stu-id="c5414-114">With the ability to customize products to customer requests, you can minimize waste by making sure that work and system changes only occur when it is necessary.</span></span>  

<span data-ttu-id="c5414-115">Du kan oppnå det ved å kombinere metoden for lagertrekk bakover og rutekoblingskoder slik at lagertrekksantallet for hver operasjon er proporsjonalt med den faktiske avgangen for den fullførte operasjonen.</span><span class="sxs-lookup"><span data-stu-id="c5414-115">You can achieve that by combining backward flushing method and routing link codes so that the quantity that is flushed for each operation is proportional to the actual output of the finished operation.</span></span> <span data-ttu-id="c5414-116">For varer som er definert med trekkmetoden Bakover, er standard virkemåte å beregne og bokføre komponentforbruk når du endrer statusen for en frigitt produksjonsordre til **Ferdig**.</span><span class="sxs-lookup"><span data-stu-id="c5414-116">For items that are set up with backward flushing method, the default behavior is to calculate and post component consumption when you change the status of a released production order to **Finished**.</span></span> <span data-ttu-id="c5414-117">Hvis du også definerer rutekoblingskoder, skjer beregning og bokføring når hver operasjon er fullført, og antallet som faktisk ble forbrukt i operasjonen, bokføres.</span><span class="sxs-lookup"><span data-stu-id="c5414-117">If you also define routing link codes, then calculation and posting occurs when each operation is finished, and the quantity that was actually consumed in the operation is posted.</span></span> <span data-ttu-id="c5414-118">Hvis du vil ha mer informasjon, kan du se [Opprette ruter](production-how-to-create-routings.md).</span><span class="sxs-lookup"><span data-stu-id="c5414-118">For more information, see [Create Routings](production-how-to-create-routings.md).</span></span>
